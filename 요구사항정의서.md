## 1. 프로젝트 개요

### 1.1 목적
OpenAI API(Whisper, GPT, TTS)를 활용하여 음성 및 텍스트 입력을 자동으로 언어 감지하고 번역하는 웹 애플리케이션을 제공한다.

### 1.2 배경
다국어 환경에서의 실시간 커뮤니케이션 지원을 위해, 음성 입력부터 텍스트/음성 출력까지 통합된 번역 워크플로우가 필요하다.

### 1.3 범위
- Streamlit 기반 웹 애플리케이션
- 4가지 번역 모드 제공 (음성 번역, 텍스트 번역, 음성→텍스트, 음성→음성)
- OpenAI API 단독 의존 (Whisper, GPT-4o-mini, TTS-1)

---

## 2. 시스템 아키텍처

```
사용자 (브라우저)
    │
    ▼
Streamlit 웹 앱 (main.py)
    │
    ├── OpenAI Whisper API  (STT: 음성 → 텍스트)
    ├── OpenAI GPT-4o-mini  (언어 감지 / 번역)
    └── OpenAI TTS-1        (TTS: 텍스트 → 음성)
```

## 3. 기능 요구사항

### 3.1 공통 요구사항

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| COM-01 | 사용자는 사이드바에서 OpenAI API 키를 입력할 수 있어야 한다 | 필수 |
| COM-02 | 환경변수 `OPENAI_API_KEY`가 설정된 경우 API 키를 자동으로 불러와야 한다 | 필수 |
| COM-03 | API 키가 없을 경우 기능 사용을 차단하고 안내 메시지를 표시해야 한다 | 필수 |
| COM-04 | API 인증 오류, API 오류, 일반 오류를 각각 구분하여 사용자에게 안내해야 한다 | 필수 |
| COM-05 | 동일한 입력에 대해 재처리를 방지하기 위해 세션 캐시를 활용해야 한다 | 권장 |

### 3.2 자동 번역 방향 규칙

| 입력 언어 | 출력 언어 |
|-----------|-----------|
| 영어 (English) | 한국어 (Korean) |
| 한국어 및 그 외 모든 언어 | 영어 (English) |

### 3.3 기능 1 — 음성 번역 (🎵 음성 번역)

| ID | 요구사항 |
|----|----------|
| F1-01 | 사용자는 브라우저 마이크를 통해 음성을 녹음할 수 있어야 한다 |
| F1-02 | 마이크 버튼 클릭으로 녹음을 시작하고, 재클릭 또는 3초 침묵 시 자동 종료되어야 한다 |
| F1-03 | 녹음된 음성을 OpenAI Whisper API로 텍스트 변환(STT)해야 한다 |
| F1-04 | Whisper가 감지한 언어를 기반으로 번역 대상 언어를 자동 결정해야 한다 |
| F1-05 | 원본 텍스트와 번역 텍스트를 2컬럼 레이아웃으로 나란히 표시해야 한다 |
| F1-06 | 감지된 언어명을 원본 텍스트 헤더에 표시해야 한다 |
| F1-07 | 원본 텍스트와 번역 텍스트를 각각 `.txt` 파일로 다운로드할 수 있어야 한다 |
| F1-08 | 처리 중 각 단계(STT, 번역)의 진행 상태를 스피너로 표시해야 한다 |

### 3.4 기능 2 — 텍스트 번역 (✏️ 텍스트 번역)

| ID | 요구사항 |
|----|----------|
| F2-01 | 사용자는 텍스트 입력창에 번역할 텍스트를 직접 입력할 수 있어야 한다 |
| F2-02 | "번역하기" 버튼 클릭 시 번역이 실행되어야 한다 |
| F2-03 | GPT-4o-mini를 이용해 입력 텍스트의 언어를 감지해야 한다 |
| F2-04 | 감지된 언어에 따라 번역 대상 언어를 자동 결정해야 한다 |
| F2-05 | 원본 텍스트와 번역 결과를 2컬럼 레이아웃으로 나란히 표시해야 한다 |
| F2-06 | 번역 텍스트를 `.txt` 파일로 다운로드할 수 있어야 한다 |
| F2-07 | 빈 텍스트 입력 시 경고 메시지를 표시해야 한다 |

### 3.5 기능 3 — 음성 → 텍스트 (🔊 음성 → 텍스트)

| ID | 요구사항 |
|----|----------|
| F3-01 | 사용자는 브라우저 마이크를 통해 음성을 녹음할 수 있어야 한다 |
| F3-02 | 녹음된 음성을 Whisper API로 STT 변환해야 한다 |
| F3-03 | Whisper가 감지한 언어를 기반으로 번역 대상 언어를 자동 결정해야 한다 |
| F3-04 | 번역된 텍스트를 메인 영역에 크게 표시해야 한다 |
| F3-05 | 감지된 언어와 번역 언어를 상단에 표시해야 한다 |
| F3-06 | 번역 텍스트를 `.txt` 파일로 다운로드할 수 있어야 한다 |
| F3-07 | 원본 음성 텍스트(STT 결과)는 접기/펼치기(expander)로 확인할 수 있어야 한다 |

### 3.6 기능 4 — 음성 → 음성 (🗣️ 음성 → 음성)

| ID | 요구사항 |
|----|----------|
| F4-01 | 사용자는 브라우저 마이크를 통해 음성을 녹음할 수 있어야 한다 |
| F4-02 | 녹음된 음성을 Whisper API로 STT 변환해야 한다 |
| F4-03 | Whisper가 감지한 언어를 기반으로 번역 대상 언어를 자동 결정해야 한다 |

| F4-05 | 한국어 출력 시 `nova` 음성, 영어 출력 시 `alloy` 음성을 사용해야 한다 |
| F4-06 | 번역된 음성이 자동 재생되어야 한다 |
| F4-07 | 번역된 음성을 `.mp3` 파일로 다운로드할 수 있어야 한다 |
| F4-08 | 원본 음성 텍스트와 번역 텍스트를 각각 접기/펼치기로 확인할 수 있어야 한다 |

## 5. 외부 인터페이스 요구사항

### 5.1 사용 API

| API | 모델 | 용도 |
|-----|------|------|
| OpenAI Whisper | `whisper-1` | 음성 → 텍스트 변환 및 언어 감지 |
| OpenAI Chat Completions | `gpt-4o-mini` | 텍스트 언어 감지 및 번역 |
| OpenAI TTS | `tts-1` | 텍스트 → 음성 변환 |

### 5.2 지원 오디오 포맷 (입력)
`mp3`, `wav`, `m4a`, `webm`, `mp4`, `mpeg`, `mpga`, `oga`

### 5.3 출력 포맷
- 텍스트: `.txt` (UTF-8)
- 음성: `.mp3`

## 9. 제약 사항

- OpenAI API 키 보유 및 충분한 API 크레딧 필요
- 마이크 녹음 기능은 HTTPS 또는 localhost 환경에서만 브라우저 허용
- Whisper 언어 감지는 입력 음성 품질에 따라 정확도가 달라질 수 있음
- TTS 출력 언어는 한국어/영어만 지원 (번역 방향 규칙에 의해 결정)
